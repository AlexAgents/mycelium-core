# Вкладка Admin

Вкладка Admin используется ролью **Администратор** для подготовки
и управления голосованием.

---

## Раздел 1 — Контракт

Управление аутентификацией администратора и развёртыванием контракта.

| Поле / Кнопка | Назначение |
|---|---|
| **Admin Private Key** | Приватный ключ владельца контракта (кошелёк, который развернёт и будет владеть контрактом). Автозаполнение из `.env` в dev-режиме. Только для чтения после деплоя. |
| **Иконка глаза (Show / Hide)** | Переключение маски пароля на поле ключа. |
| **Баланс** | Текущий ETH-баланс admin-адреса. Компактный формат (4 знака для обычных значений, K/M/B/T для больших, научная нотация для >1e15). |
| **Dev: X ETH** | Дополнительный лейбл с балансом dev-аккаунта Geth (виден только в dev-режиме). |
| **Fund from Dev** | Переводит 100 ETH с dev-аккаунта Geth на admin-адрес. Доступно только в dev-режиме. |
| **Иконка обновления** | Перезагружает admin-баланс из цепи. |
| **Развернуть VotingCore** | Компилирует `contracts/VotingCore.sol` и разворачивает на локальной цепи. Операция в один клик. |
| **Адрес контракта** | Адрес развёрнутого контракта. Можно выделять для копирования. |
| **Текущая стадия** | SETUP / ACTIVE / FINISHED (текстовый лейбл). |

После успешного деплоя:

- Поле admin key становится только для чтения (заблокировано до Новой сессии).
- Разделы кандидатов и избирателей разблокируются.
- Отображение стадии показывает `SETUP`.

---

## Раздел 2 — Кандидаты

Подготовьте кандидатов локально, затем отправьте в сеть одним батчем.

| Поле / Кнопка | Назначение |
|---|---|
| **Имя** | Имя кандидата. Обязательно. |
| **Партия** | Политическая принадлежность. Обязательно. |
| **Ethereum-адрес** | On-chain адрес кандидата. Обязательно. |
| **Сгенерировать (иконка магической палочки)** | Автоматически генерирует случайный валидный Ethereum-адрес. |
| **Добавить** | Добавляет кандидата в таблицу. Проверяет формат адреса и дубликаты. |
| **Таблица кандидатов** | Показывает добавленных кандидатов (Имя / Партия / Адрес). Строки выбираются. |
| **Удалить** | Удаляет выбранную строку. |
| **Зарегистрировать в сети** | Отправляет по одной транзакции `addCandidate` на каждого кандидата. Заблокировано при `< 2` кандидатах. |

Ограничения:

- Максимум 10 кандидатов (требование контракта).
- Минимум 2 кандидата для запуска голосования.
- После "Зарегистрировать в сети": поля кандидатов блокируются до Новой сессии.

## Раздел 3 — Избиратели

Генерация, импорт, финансирование и whitelist избирателей. **Поддерживается
перетаскивание JSON** — бросьте файл `voters.json` в любое место этого
группбокса.

### Строка 1: Кнопки действий

| Кнопка | Назначение |
|---|---|
| **Импорт** | Открывает диалог файла, загружает JSON со списком избирателей. |
| **Экспорт** | Сохраняет избирателей текущей сессии в JSON. Показывает предупреждение безопасности перед записью (файл содержит приватные ключи). |
| **Сгенерировать** | Генерирует N тестовых избирателей (пары ключей). Поле "Количество" задаёт N (1–1000). |
| **Финансировать** | Отправляет ETH всем избирателям сессии. Поле "ETH" задаёт сумму на избирателя (по умолчанию 0.01). |

### Строка 2: Статистика

- **Загружено** — количество избирателей в памяти сессии.
- **В whitelist** — количество добавленных в whitelist в сети.
- **Профинансированы** — количество успешно профинансированных.

### Строка 3: Whitelist

**Добавить избирателей в whitelist** — отправляет одну транзакцию
`addVotersBatch` со всеми адресами сессии.

### Мини-лог

Последние строки лога операций (компиляция, деплой, финансирование,
whitelist).

---

## Раздел 4 — Управление стадией

Две большие кнопки:

- **Начать голосование** — вызывает `startVoting()`. Требует:
  - Контракт развёрнут.
  - Минимум 2 кандидата зарегистрировано.
  - Минимум 1 избиратель в whitelist (UX-проверка; контракт допускает пустой).

  Стадия меняется на `ACTIVE`. Вкладка Голосование становится активной.

- **Завершить голосование** — вызывает `finishVoting()`. Стадия меняется
  на `FINISHED`. Вкладка Голосование блокируется. Режимы Final/Full
  аудита разблокируются.

---

## Типичный сценарий

1. Развернуть контракт.
2. Добавить 2+ кандидатов → Зарегистрировать в сети.
3. Сгенерировать избирателей → Финансировать → Добавить в whitelist.
4. Начать голосование.
5. Переключиться на вкладку Голосование.
6. После голосования: вернуться сюда → Завершить голосование.
7. Переключиться на вкладку Аудит.
8. Провести аудит. Проверить результат.
9. Экспортировать результаты голосования и результаты аудита.

## Ошибки и диалоги

Все actionable-ошибки (недостаточно средств, конфликт nonce, revert
контракта) открывают **ErrorDialog** с переведённым сообщением и
опциональной кнопкой действия (например, "Пополнить счёт из admin-кошелька?").
Полный список распознаваемых паттернов ошибок описан в `ErrorParser`
в [API Reference](../api/index.md).
