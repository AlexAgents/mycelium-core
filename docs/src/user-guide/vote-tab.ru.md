# Вкладка Голосование

Вкладка Голосование используется ролью **Избиратель** для аутентификации
и подачи голоса. Также содержит режим **Массовое голосование** для
нагрузочного тестирования.

---

## Раздел 1 — Аутентификация

| Поле / Кнопка | Назначение |
|---|---|
| **Приватный ключ** | Приватный ключ избирателя (hex-строка с префиксом `0x` или без). По умолчанию скрыт маской. |
| **Иконка глаза (Show / Hide)** | Переключение маски пароля. |
| **Загрузить JSON** | Загружает ключ первого избирателя из файла `voters.json`. |

### Сетка статусов

Пять блоков статусов обновляются **автоматически** через ~600 мс после
последнего нажатия (debounced асинхронный запрос):

- **Адрес** — вычислен из приватного ключа (обрезан для отображения).
- **В whitelist** — `YES` (зелёный) / `NO` (красный).
- **Голосовал** — `NO` (зелёный) / `YES` (красный).
- **Стадия** — текущая стадия контракта с цветовой индикацией.
- **Баланс** — ETH-баланс избирателя.

> **Примечание (v1.0.1):** до рефакторинга UI зависал на ~50–200 мс
> при каждом нажатии из-за трёх синхронных RPC-вызовов. Запросы
> статуса теперь выполняются в выделенном потоке `VoterStatusWorker`.

---

## Раздел 2 — Выбор кандидата

- **Загрузить кандидатов** — получает список зарегистрированных
  кандидатов из контракта. Требуется после регистрации на вкладке
  Admin, если автозагрузка не сработала.

- **Список кандидатов** — radio-кнопки. Каждая строка: Имя | Партия | Адрес.

Можно выбрать ровно одного кандидата.

---

## Раздел 3 — Действие

**ПРОГОЛОСОВАТЬ** — отправляет транзакцию `castVote(candidate)`,
подписанную введённым приватным ключом.

Перед отправкой приложение выполняет **5-уровневую предпроверку**
(`AppController.precheck_vote`):

1. Ключ валиден?
2. Контракт развёрнут?
3. Адрес в whitelist?
4. Адрес уже голосовал?
5. Баланса достаточно на газ (>= 0.0003 ETH)?

Если любая проверка не пройдена, пользователь получает переведённый
диалог-предупреждение, а транзакция **не** отправляется (экономия газа).
См. диаграмму активности [Валидация перед голосованием](../diagrams/activity/precheck-vote.ru.md).

---

## Раздел 4 — Квитанция

После успешного голосования:

- **TX Hash** — хэш транзакции, кнопка копирования.
- **QR-квитанция** — изображение 140x140 px с TX hash, адресом
  избирателя, адресом кандидата, номером блока, ID сессии.
  Сохранение через **Сохранить QR**.

Поле приватного ключа очищается автоматически.

## Раздел 5 — Массовое голосование (Тестирование)

Для нагрузочного тестирования и демо: автоматическая подача случайных
голосов от множества избирателей.

Два источника:

- **Из JSON-файла** — выберите файл `voters.json`.
- **Из сессии** — использовать избирателей, сгенерированных на
  вкладке Admin.

Конвейер на каждого избирателя:

1. Предпроверка (ключ валиден, whitelist, голосовал, баланс).
2. Пропуск, если проверка не пройдена — причина в лог.
3. Случайный выбор кандидата.
4. Подача голоса.
5. Запись в лог массового голосования.

После завершения тост показывает: `Проголосовало N, пропущено M, ошибок K`.

Массовое голосование **не** останавливается при единичной ошибке;
обработка продолжается со следующего избирателя.

## Блокировка по стадиям

Вкладка Голосование блокирует операции в зависимости от стадии:

| Стадия | Поведение |
|---|---|
| **Контракт не развёрнут** | Диалог "Сначала разверните контракт" блокирует голосование. |
| **SETUP** | Диалог "Голосование ещё не начато" блокирует. |
| **ACTIVE** | Голосование разрешено. |
| **FINISHED** | Диалог "Голосование завершено" блокирует. |

---

## Ошибки

Все ошибки транзакций (недостаточно газа, двойное голосование, не в
whitelist, конфликт nonce) переводятся через `ErrorParser` и показываются
в **ErrorDialog**. Ключи локализации: `err.parser.*` (см. i18n проекта).
