# Вкладка Логи

Вкладка Логи отображает файл активного лога сессии в реальном времени.
Полезна всем ролям для отладки и проверки поведения.

---

## Что отображается

Содержимое `data/logs/active/session.log` — файла, в который приложение
пишет логи в реальном времени.

Лог пересоздаётся при каждом запуске приложения. Логи предыдущих
сессий автоматически архивируются в `data/logs/archive/<session_id>/session.log`.

---

## Панель инструментов (слева направо)

| Кнопка | Назначение |
|---|---|
| **Обновить** | Перечитывает лог с диска. Необходимо после длительных операций для просмотра последних записей. |
| **Копировать всё** | Копирует весь лог в буфер обмена. |
| **Сохранить как** | Сохраняет текущее представление в файл `.log` или `.txt`. |
| **Очистить** | Очищает только окно просмотра (файл **не** удаляется). |

Панель (справа):

| Элемент | Назначение |
|---|---|
| **Поле поиска** | Live-фильтр — показывает только строки, содержащие подстроку (без учёта регистра). |
| **Чекбокс автопрокрутки** | Если включён, окно прокручивается вниз при каждом обновлении. |

---

## Окно просмотра

QTextEdit только для чтения с моноширинным шрифтом. Перенос слов
отключён (одна запись лога на строку для читаемости).

---

## Нижняя инфо-строка

| Поле | Описание |
|---|---|
| **Строк** | Количество отображаемых строк (после фильтра поиска). |
| **Размер** | Размер файла лога на диске (формат: Б / КБ / МБ). |
| **Путь** | Полный путь к `session.log`. Можно выделять для копирования. |

---

## Что логируется

Приложение пишет в лог (через Python `logging`):

- Жизненный цикл процесса Geth (старт, стоп, крах).
- Результат компиляции и адрес контракта после деплоя.
- Каждое действие администратора (добавление кандидата, whitelist batch, смена стадии).
- Каждый `cast_vote` с TX hash.
- Сводку массового голосования.
- Сводку запуска аудита.
- Ошибки и необработанные исключения (с трассировкой стека).
- Жизненный цикл сессии (создание, архивация).

---

## Что НЕ логируется

`_SecretFilter` автоматически вырезает:

- Строки, содержащие `private_key`, `priv_key`, `mnemonic`, `seed phrase`, `secret`.
- Всё, что соответствует regex `0x[a-fA-F0-9]{64}` (паттерн приватного ключа).

Такие записи заменяются на `[REDACTED — secret suppressed]`.

---

## Архив

При нажатии **Новая сессия** в хедере текущий лог архивируется:

```text
data/logs/archive/<session_id>/session.log
```

Активный файл лога очищается и продолжается уже для новой сессии.
