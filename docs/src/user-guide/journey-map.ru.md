# Карта пользовательского пути

Сквозной путь через полный цикл голосования в **MYCELIUM CORE**.
Три актёра, пять фаз, одна сессия.

---

## Обзор пути

```text
Фаза 1          Фаза 2          Фаза 3          Фаза 4          Фаза 5
ПОДГОТОВКА  -->  НАСТРОЙКА  -->  ГОЛОСОВАНИЕ -->  АУДИТ      -->  АРХИВАЦИЯ
(Админ)          (Админ)         (Избиратель)     (Аудитор)       (Админ)
```

---

## Фаза 1: Подготовка (Администратор)

| Шаг | Действие | Ответ системы | Эмоция |
|:---|:---|:---|:---|
| 1.1 | Запуск приложения | Geth стартует, RPC подключается, сессия создана | Нейтрально |
| 1.2 | Ввод admin key | Ключ провалидирован, баланс показан | Уверенность |
| 1.3 | Fund from Dev | 100 ETH переведено, баланс обновлён | Удовлетворение |
| 1.4 | Деплой контракта | Контракт скомпилирован и развёрнут, адрес показан | Достижение |

**Точки контакта:** вкладка Admin, статус-бар в подвале, тост-уведомления

**Болевые точки:** Запуск Geth может занять 2-5 секунд. Конфликт порта 8545 при повторном запуске.

**Возможности:** Опция авто-деплоя в диалоге "Новая сессия".

---

## Фаза 2: Настройка (Администратор)

| Шаг | Действие | Ответ системы | Эмоция |
|:---|:---|:---|:---|
| 2.1 | Добавление кандидатов (2+) | Кандидаты в локальной таблице | Сосредоточенность |
| 2.2 | Регистрация в сети | TX на каждого, поля заблокированы | Уверенность |
| 2.3 | Генерация избирателей | Пары ключей созданы, счётчик обновлён | Эффективность |
| 2.4 | Финансирование | ETH распределён, прогресс-бар активен | Терпение |
| 2.5 | Добавление в whitelist | Пакетная TX отправлена, счётчик обновлён | Готовность |
| 2.6 | Начало голосования | Стадия меняется на ACTIVE | Достижение |

**Точки контакта:** вкладка Admin (все 4 секции), диалоги подтверждения, прогресс-бар

**Болевые точки:** Финансирование 1000+ избирателей медленное (последовательный nonce). TX whitelist может упасть на больших батчах.

**Возможности:** Оптимизация пакетного финансирования. Оценка времени операции.

## Фаза 3: Голосование (Избиратель)

| Шаг | Действие | Ответ системы | Эмоция |
|:---|:---|:---|:---|
| 3.1 | Ввод приватного ключа | Адрес вычислен, сетка статусов обновлена | Осторожность |
| 3.2 | Проверка статуса | В whitelist: ДА, Голосовал: НЕТ | Уверенность |
| 3.3 | Загрузка кандидатов | Появляются radio-кнопки | Выбор |
| 3.4 | Выбор кандидата | Radio-кнопка отмечена | Решимость |
| 3.5 | Подача голоса | TX отправлена, QR-квитанция показана | Удовлетворение |
| 3.6 | Сохранение квитанции | QR PNG сохранён на диск | Безопасность |

**Точки контакта:** вкладка Vote, сетка статусов, QR-квитанция, тост

**Болевые точки:** Предупреждение SEC-06 при вводе ключа может запутать. Ввод приватного ключа ощущается рискованным.

**Возможности:** Drag-and-drop файла ключа. Понятные пояснения статусов.

---

## Фаза 4: Аудит (Аудитор)

| Шаг | Действие | Ответ системы | Эмоция |
|:---|:---|:---|:---|
| 4.1 | Открытие вкладки Аудит | Таблица результатов пуста, режимы доступны | Исследование |
| 4.2 | Выбор Полного аудита | Описание режима показано | Сосредоточенность |
| 4.3 | Запуск аудита | 6 SEC-проверок выполнены, таблица заполнена | Аналитика |
| 4.4 | Просмотр результатов | Бейджи PASSED/FAILED, победитель показан | Информированность |
| 4.5 | Экспорт отчёта | JSON/CSV сохранён на диск | Задокументировано |

**Точки контакта:** вкладка Аудит, цвета StatusBadge, кнопки экспорта

**Болевые точки:** SEC-03 может вернуть WARNING если события стадий недоступны. Термины аудита могут быть незнакомы.

**Возможности:** Тултипы с пояснениями проверок. Визуальное сравнение между запусками аудита.

## Фаза 5: Архивация (Администратор)

| Шаг | Действие | Ответ системы | Эмоция |
|:---|:---|:---|:---|
| 5.1 | Просмотр результатов | Победитель подтверждён на вкладке Аудит | Удовлетворение |
| 5.2 | Экспорт результатов | JSON/CSV сохранён | Задокументировано |
| 5.3 | Новая сессия | Логи заархивированы, UI сброшен | Готовность |
| 5.4 | (Опционально) Сброс | Chain data удалены, Geth перезапущен | Чистота |

**Точки контакта:** экспорт вкладки Аудит, кнопки в хедере, диалоги подтверждения

**Болевые точки:** Сброс может упасть если Windows держит файловые блокировки. Инструкции ручной очистки могут запутать.

**Возможности:** Автоэкспорт при завершении сессии. Отчёт "полный цикл" одним кликом.

## Метрики пути

| Метрика | Цель | Факт |
|:---|:---|:---|
| Время от запуска до первого голоса | < 5 минут | ~3 минуты |
| Шагов для деплоя + настройки | 6 кликов + ввод | 6 |
| Шагов для одного голоса | 4 клика + ввод ключа | 4 |
| Шагов для полного аудита | 2 клика | 2 |
| Полный цикл (5 избирателей, 3 кандидата) | < 10 минут | ~7 минут |
