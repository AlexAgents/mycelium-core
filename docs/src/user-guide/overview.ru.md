# Руководство пользователя — Общие концепции

Добро пожаловать в руководство пользователя **MYCELIUM CORE**. Приложение функционирует как изолированная среда-песочница для демонстрации целостной системы голосования на блокчейне.

---

## Концепция: Одна сессия — Одно голосование
Каждый цикл голосования в MYCELIUM CORE привязан к изолированному одноразовому смарт-контракту. Повторное использование контракта запрещено на уровне Solidity. Это означает:

- Запуск новой сессии голосования требует **развёртывания нового экземпляра контракта**.
- Жизненный цикл смарт-контракта движется строго вперёд: `Setup -> Active -> Finished`.
- После перехода в состояние `Finished` все данные замораживаются навсегда, делая возможным достоверный криптографический аудит.

## Роли пользователей

1. **Администратор:** Деплоит контракт, регистрирует кандидатов, импортирует и заносит избирателей в белый список, а также управляет стадиями.
2. **Избиратель:** Авторизуется по приватному ключу, просматривает список кандидатов и отправляет подписанную транзакцию для голосования.
3. **Аудитор:** Выполняет независимые проверки для верификации безопасности и целостности данных в блокчейне.
