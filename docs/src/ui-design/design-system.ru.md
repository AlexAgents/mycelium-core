# Дизайн-система UI

Интерфейс MYCELIUM CORE следует строгому, структурированному и
современному языку визуального оформления, адаптированному для
PyQt6 с поддержкой тем.

---

## Принципы проектирования

1. **Структурированность над хаосом:** Использование карточек
   (`QGroupBox`) и вертикальных отступов вместо плотных рамок
   или бесконечных разделительных линий.

2. **Консистентность тем:** Каждый компонент, значок статуса и
   кастомный диалог должны динамически адаптироваться к тёмному
   и светлому режимам.

3. **Никакого чёрного в светлой теме:** Чистый чёрный цвет
   (`#000000`) полностью запрещён в светлой теме. Он заменён
   палитрой глубокого синего (`#1f3a68`) и навигационных
   оттенков (`#3d5a80`).

4. **Смысловая палитра:** Цвета представляют семантические состояния:
   - **Зелёный** для успеха, активных стадий и действий голосования.
   - **Синий** для вторичных процессов, деплоя и отображения данных.
   - **Оранжевый/Жёлтый** для предупреждений, стадии настройки и завершения.
   - **Красный** для критических действий, ошибок и проваленных проверок.
   - **Фиолетовый** для завершённого состояния и глобальных системных нот.

5. **Два оттенка зелёного -- намеренно:** `#1a7f37` / `#3fb950` для
   бейджей и кнопок, `#2da44e` для Toast-уведомлений. Разные уровни
   яркости обеспечивают контраст на разных фонах.

6. **Тосты не зависят от темы:** Цвета Toast-уведомлений заданы через
   inline `setStyleSheet()` в Python-коде, а не через QSS-файлы.
   Они одинаковы в обеих темах.

## Иерархия компонентов (Atomic Design)

UI следует упрощённому подходу Atomic Design:

| Уровень | Примеры | Расположение |
|:---|:---|:---|
| **Атомы** | `StatusBadge`, `Toast`, подписи полей, пины | `src/ui/widgets/` |
| **Молекулы** | `_StatusBlock` (подпись + значение), строка ключа (поле + глаз) | Inline в файлах вкладок |
| **Организмы** | Секции `QGroupBox` (Контракт, Кандидаты, Избиратели, Стадия) | Методы `_build_*_section()` |
| **Шаблоны** | Макеты вкладок (`AdminTab`, `VoteTab`, `AuditTab`, `LogsTab`) | `src/ui/tabs/` |
| **Страницы** | `MainWindow` с хедером, вкладками и статус-баром | `src/ui/main_window.py` |

## Зона Drag-and-Drop

Секция Избиратели на вкладке Админ реализует зону перетаскивания
для импорта JSON-файлов.

### Визуальные состояния

| Состояние | Стиль границы |
|:---|:---|
| Обычное | `1px solid #c4d3e8` (светлая) / `1px solid #30363d` (тёмная) |
| При перетаскивании | `2px dashed #0969da` (светлая) / `2px dashed #388bfd` (тёмная) |

Зона реализована как подкласс `QGroupBox` (`_VoterDropZone`) с
`setAcceptDrops(True)`. Свойство `dragOver` переключается через
`setProperty("dragOver", True/False)` и стилизуется через QSS:
`QGroupBox[dragOver="true"]`.

Принимаются только файлы с расширением `.json`.

**Код:** `src/ui/tabs/admin_tab.py`
