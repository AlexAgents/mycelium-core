# Макеты Figma и Wireframes

Документ описывает структуру, содержание и систему аннотаций
Figma-файла дизайна **MYCELIUM CORE**.

---

## Файл дизайна Figma

Полные макеты пользовательского интерфейса с аннотациями доступны по ссылке:

**[Открыть в Figma (только для просмотра)](https://www.figma.com/design/PWzJmLP7TrrbjcL6F85KoU/mycelium-core)**

Файл содержит 4 макета вкладок, 5 типов диалоговых окон, вариативные компоненты,
и 53 аннотации. Подробности структуры см. ниже.

### Структура файла

Figma-файл содержит все UI-макеты для светлой темы, организованные
в одну горизонтальную линию слева направо:

```text
[Легенда] -- [Admin Tab] -- [Vote Tab] -- [Audit Tab] -- [Logs Tab] -- [Header+Footer] -- [Сообщения] -- [Вариативные]
```

Расстояние между блоками: 400px. Направление чтения: слева направо,
совпадает с workflow приложения и порядком вкладок.

### Содержимое блоков

| Блок | Содержимое | Кол-во нот |
|:---|:---|:---|
| Легенда | Справочник цветов, стиль пинов, правила расположения | -- |
| Admin Tab | Полный макет + 20 аннотационных нот | 20 |
| Vote Tab | Полный макет + 10 аннотационных нот | 10 |
| Audit Tab | Полный макет + 8 аннотационных нот | 8 |
| Logs Tab | Полный макет + 5 аннотационных нот | 5 |
| Header + Footer | Макеты хедера и статус-бара + 3 ноты | 3 |
| Сообщения | 5 типов диалогов + 3 аннотационные ноты | 3 |
| Вариативные | StatusBadge, Toast, ProgressBar, LogBox + 4 ноты | 4 |

**Итого: 53 аннотационные ноты по всем блокам.**

## Соглашение по именованию слоёв

Каждый блок следует единой структуре именования:

```text
{tab}-tab-full/
  {tab}-tab-alexagents    -- фрейм макета
  {tab}-tab-notes         -- контейнер нот
```

Пример:

```text
admin-tab-full/
  admin-tab-alexagents
  admin-tab-notes
```

Вспомогательные блоки:

```text
header-footer-full/
  header-footer
  header-footer-notes
msg-full
var-full
legend
```

## Система аннотаций

### Формат ноты

Каждая нота представляет собой карточку со следующей структурой:

```text
[SYS-NOTE-XXX] Заголовок
Версия: 1.0.1  Статус: Актуально  Дата: 2026-05-20  Автор: AlexAgents

Привязка к интерфейсу:
  WidgetClass.widget_name

Суть:
  1-2 предложения о поведении.

Связи:
  ADR-XXX  |  FR-XXX  |  src/path/to/file.py

Ограничение: (если есть)
Комментарий: (если есть)

MYCELIUM CORE | Обновлено: 2026-05-20
```

### Цвета обводки нот

Ноты используют цветные обводки для обозначения категории:

| Цвет | Hex | Категория | Стиль обводки |
|:---|:---|:---|:---|
| Синий | `#0969da` | Поля ввода, отображение данных, таблицы | Со стороны таба |
| Зелёный | `#1a7f37` | Кнопки действий, операции, workers | Со стороны таба |
| Оранжевый | `#bf6a02` | Стадии, ограничения, предупреждения | Полная обводка |
| Фиолетовый | `#6e40c9` | Глобальные ноты, система целиком | Полная обводка |

Синие и зелёные ноты имеют обводку только со стороны, обращённой
к макету, указывая направление. Оранжевые и фиолетовые ноты имеют
полную обводку со всех четырёх сторон, указывая на системную область.

### Расположение нот вокруг макета

- **Первая нота:** по центру над макетом таба
- **Последняя нота:** по центру под макетом таба
- **Нечётные ноты** (01, 03, 05...): справа от таба
- **Чётные ноты** (02, 04, 06...): слева от таба

```text
              [НОТА-001]
+---------+               +---------+
|НОТА-002 |  +---------+  |НОТА-003 |
|(чётная) |  |         |  |(нечётн.)|
+---------+  |   TAB   |  +---------+
             |  МАКЕТ   |
+---------+  |         |  +---------+
|НОТА-004 |  |         |  |НОТА-005 |
|(чётная) |  +---------+  |(нечётн.)|
+---------+               +---------+
            [НОТА-ПОСЛЕДНЯЯ]
```

## Система пинов

Пины -- пронумерованные маркеры на макете, указывающие к какому
элементу относится нота.

### Стиль пина

- **Форма:** круг, 16x16px
- **Заливка:** `#F5C844`
- **Обводка:** 2px solid `#1f3a68`
- **Текст:** `#1f3a68`, 10px bold, по центру
- **Слой:** поверх всех фреймов (не вложен во фрейм элемента)
- **Положение:** свободно перемещаемый, касается элемента

### Правила размещения пинов

- **Нечётные пины** (03, 05, 07): справа от элемента
- **Чётные пины** (02, 04, 06): слева от элемента
- **Глобальные ноты** (напр. ADM-020): пин внутри фрейма,
  охватывающего весь контент таба

Номер пина совпадает с номером ноты. Пин 03 на макете
соответствует ноте [SYS-NOTE-ADM-003] в столбце нот.

## Спецификации макетов

### Размеры окна

- **Минимальный размер:** 1100x700px
- **Размер макета:**px
- **Высота хедера:** 44px (фиксированная)
- **Высота статус-бара:** 28px (фиксированная)
- **Контент вкладки:** оставшаяся высота, скроллируемый

### Тема

Все макеты используют исключительно **светлую тему**. Значения
цветов соответствуют палитре светлой темы, задокументированной
в [Цветовая палитра](./color-palette.ru.md).

### Макеты диалогов

Пять типов диалогов задокументированы в блоке Сообщения:

1. **Новая сессия** -- подтверждение с checkbox авто-деплоя
2. **Массовое голосование** -- подтверждение перед пакетным голосованием
3. **Финансирование** -- подтверждение с расчётом суммы
4. **Выход** -- подтверждение выхода
5. **Сброс блокчейна** -- деструктивное действие с опцией удаления логов

## Вариативные компоненты

Блок Вариативные документирует переиспользуемые компоненты
со всеми состояниями и вариантами:

- **StatusBadge:** 11 вариантов ключей в двух группах
  (Аудит/Безопасность и Стадия/Подключение)
- **Toast:** 4 типа (success, error, warning, info) с примерами
- **ProgressBar:** 4 состояния (0%, 30%, 60%, 100%)
- **LogBox:** 3 примера содержимого (деплой, кандидаты, избиратели)
