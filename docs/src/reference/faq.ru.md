# Часто задаваемые вопросы

Ответы на частые вопросы о настройке, использовании и архитектуре
**MYCELIUM CORE**.

---

### Почему блоки начинаются с 0 после каждого перезапуска?

Geth в режиме `--dev` создаёт эфемерную цепь — состояние хранится
в RAM, на диске только заголовки. Это **архитектурное решение**
(см. [ADR-003](../architecture/decisions/adr-003-ephemeral-chain.ru.md)),
а не баг. Одна сессия = одно голосование.

---

### Почему admin-баланс 0 после деплоя контракта?

В dev-режиме баланс есть только у **dev-аккаунта Geth**, не у вашего
admin-ключа. Нажмите **Fund from Dev** (иконка ладони с долларом рядом
с балансом) — она переведёт 100 ETH с dev-аккаунта на ваш admin-адрес.

---

### Что делать, если порт 8545 занят?

При старте приложение пытается завершить зависшие процессы `geth.exe`.
Если порт всё ещё занят:

```bash
taskkill /F /IM geth.exe /T
```

Затем перезапустите приложение. Если проблема сохраняется — смените
`RPC_PORT` в файле `.env` на свободный порт (например, `8546`).

---

### Где хранятся логи прошлых сессий?

В `data/logs/archive/<session_id>/session.log`. Логи автоматически
архивируются при создании новой сессии или запуске приложения.

---

### Можно ли использовать в реальных выборах?

**Нет.** MYCELIUM CORE — это **демонстрационный sandbox** для
магистерской исследовательской работы. Для реальных выборов требуются
аппаратные подписи, криптографическая анонимность, независимая
сертификация и многое другое. См.
[Известные ограничения](../security/known-limitations.ru.md).

---

### Почему PyQt6, а не PySide6?

См. [ADR-001](../architecture/decisions/adr-001-pyqt6-choice.ru.md). PyQt6
выбран из-за лучшей совместимости с PyInstaller и `qtawesome` на Windows.

---

### Как запустить тесты?

```bash
cd mycelium-core
python -m pytest -v
```

Все тесты — в директории `tests/`. Для интеграционных тестов (требуется
бинарник Geth):

```bash
python -m pytest -m integration -v
```

Подробности — в [Руководстве по тестированию](../development/testing.ru.md).

---

### Компилятор Solidity скачивается автоматически?

Да. `py-solc-x` автоматически скачивает бинарник компилятора при первом
деплое. Убедитесь в наличии интернет-соединения при первом запуске.
Последующие запуски используют кешированную версию.

---

### Как переключить тему?

Нажмите **кнопку переключения темы** (иконка adjust) в хедере. Выбор
сохраняется в `app.cfg` и действует между перезапусками.

---

### Как сменить язык интерфейса?

Используйте **комбобокс EN/RU** в хедере. Большинство надписей
обновляется мгновенно. Некоторые требуют перезапуска приложения —
об этом сообщит всплывающее уведомление.

---

### Что такое `_SecretFilter` в логгере?

Фильтр безопасности, который автоматически вырезает приватные ключи и
чувствительные строки из всех записей лога. Строки, соответствующие
`0x[a-fA-F0-9]{64}` или содержащие `private_key`, `mnemonic`, `secret`,
заменяются на `[REDACTED]`.

---

### Как работает массовое голосование?

Mass Vote перебирает список избирателей и для каждого выполняет
5-уровневую предпроверку (валидность ключа, whitelist, has_voted,
баланс). Правомочные избиратели голосуют за случайного кандидата.
При неудаче проверки избиратель пропускается — обработка продолжается.
Результат: `Проголосовало N, пропущено M, ошибок K`.

---

### Почему поле admin key блокируется после деплоя?

Чтобы исключить случайную смену владельца контракта в ходе сессии.
Поле разблокируется при создании **Новой сессии**.

---

### Что произойдёт при нажатии «Сброс данных (ТЕСТ)»?

Тестовая функция из ранних версий. Система останавливает Geth, удаляет
`chain-data/active/`, при необходимости удаляет архивные логи,
перезапускает Geth и создаёт новую сессию. См.
[Диаграмму сброса блокчейна](../diagrams/sequence/reset-blockchain.ru.md).

---

### Как экспортировать результаты аудита?

На вкладке **Аудит** после запуска аудита используйте:

- **Копировать отчёт** — JSON в буфер обмена.
- **JSON** — полный отчёт в файл `.json`.
- **CSV** — структурированный отчёт в файл `.csv` (секции:
  `[Results]`, `[Winner]`, `[Audit]`).

Все три используют единый источник:
`AppController.build_full_report()`.

---

### Спишется ли реальный ETH если вставить реальный приватный ключ?

Нет. Приложение работает на локальном узле Geth `--dev` (network ID
1337) который полностью изолирован от Ethereum mainnet и всех
публичных тестовых сетей. Транзакции никогда не покидают вашу машину.

Однако вставлять реальный приватный ключ в любое desktop-приложение
опасно с точки зрения безопасности — ключ существует в памяти
процесса. Всегда используйте сгенерированные тестовые ключи. См.
[Известные ограничения](../security/known-limitations.ru.md#9-real-ethereum-accounts-and-private-keys).
