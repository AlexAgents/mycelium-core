# Глоссарий
Ключевые термины экосистемы **MYCELIUM CORE**.

---

## Блокчейн-термины
| Термин | Определение |
|---|---|
| **ABI** | Application Binary Interface — JSON-спецификация, описывающая публичные методы, события и типы данных смарт-контракта. Используется Web3 для кодирования/декодирования вызовов. |
| **Байт-код (Bytecode)** | Скомпилированное бинарное представление Solidity-контракта, развёртываемое в блокчейне. |
| **Деплой (Deploy)** | Процесс публикации скомпилированного смарт-контракта в блокчейне путём отправки creation-транзакции. |
| **ETH (Ether)** | Нативная криптовалюта сети Ethereum. Используется для оплаты газа. |
| **Газ (Gas)** | Единица измерения вычислительных усилий для выполнения операций в сети Ethereum. |
| **Geth (go-ethereum)** | Официальная реализация протокола Ethereum на Go. Используется локально в режиме `--dev`. |
| **Генезис-блок** | Первый блок (блок 0) блокчейна. В dev-режиме создаётся автоматически. |
| **JSON-RPC** | Протокол удалённого вызова процедур через JSON. Geth предоставляет его на порту 8545. |
| **Nonce** | Последовательный счётчик транзакций аккаунта. Предотвращает атаки повторного воспроизведения. |
| **RPC** | Remote Procedure Call — протокол связи между приложением и узлом Geth. |
| **Смарт-контракт** | Самоисполняемый цифровой контракт с условиями, записанными в коде и развёрнутыми в блокчейне. |
| **Квитанция транзакции** | Структура данных, возвращаемая узлом после майнинга транзакции: статус, газ, логи, номер блока. |
| **Wei** | Наименьшая единица ETH. 1 ETH = 10^18 Wei. |
| **Whitelist** | Список Ethereum-адресов, зарегистрированных в сети и имеющих право голосовать. |

## Термины приложения
| Термин | Определение |
|---|---|
| **ADR** | Architecture Decision Record — документ, фиксирующий важное архитектурное решение, его контекст и последствия. |
| **Аудит** | Процесс проверки целостности выборов путём анализа on-chain событий независимо от состояния контракта. |
| **AppController** | Класс-фасад, координирующий все доменные сервисы и предоставляющий чистые методы слою UI. |
| **BPMN** | Business Process Model and Notation — стандарт моделирования бизнес-процессов. |
| **Кандидат** | Лицо или сущность, зарегистрированная в сети для получения голосов на выборах. |
| **Dev-режим** | Режим удобства, при котором admin-ключ автозагружается из `.env`. Явно помечен как небезопасный в UI. |
| **ErrorParser** | Доменный сервис, преобразующий сырые RPC/контрактные ошибки в понятные i18n-сообщения. |
| **Фасад (Facade)** | Паттерн проектирования, предоставляющий упрощённый интерфейс к сложной подсистеме. `AppController` реализует этот паттерн. |
| **Массовое голосование** | Тестовая функция автоматической подачи случайных голосов от множества избирателей. |
| **NonceManager** | Потокобезопасная утилита для отслеживания и выдачи последовательных nonce для транзакций. |
| **Предпроверка (Precheck)** | 5-уровневый конвейер валидации перед отправкой голоса (валидность ключа, наличие контракта, whitelist, has_voted, баланс). |
| **QR-квитанция** | QR-код с TX hash, адресами избирателя и кандидата, номером блока и ID сессии. |
| **QSS** | Qt Style Sheet — CSS-подобный язык стилизации виджетов PyQt6. |
| **SEC-проверка** | Один из шести инвариантов безопасности (SEC-01 — SEC-06), проверяемых сервисом AuditService. |
| **Сессия** | Изолированный одноразовый контекст голосования, привязанный к одному экземпляру контракта. |
| **SessionContext** | Структура данных в оперативной памяти с состоянием текущей сессии (адрес контракта, кандидаты, избиратели, deploy block). |
| **SRS** | Software Requirements Specification — формальный документ требований к проекту. |
| **Стадия (Stage)** | Одна из трёх фаз жизненного цикла контракта: SETUP → ACTIVE → FINISHED. Переходы необратимы. |
| **Тост (Toast)** | Плавающее уведомление в правом нижнем углу окна. |
| **Избиратель (Voter)** | Ethereum-адрес, зарегистрированный в whitelist и имеющий право подать ровно один голос. |
| **VoteReceipt** | Типизированный DTO с TX hash, номером блока, адресами и опциональными QR-байтами. |
| **Воркер (Worker)** | Фоновая задача на базе `QThread`, выполняющая длительные операции без блокировки UI. |
| **VoterStatus** | Типизированный DTO-снимок состояния избирателя (whitelist, voted, balance, stage). Добавлен в v1.0.1 для отвязки UI от RPC. |
| **PrecheckResult** | Типизированный объект результата, возвращающий статус 5-уровневого конвейера предголосовой валидации. |
| **ParsedError** | Структурированное представление разобранной ошибки RPC или контракта, содержащее i18n-ключи для перевода в UI. |
| **ResetBlockchainWorker** | Фоновый воркер, отвечающий за безопасную остановку Geth, очистку chain-data и переинициализацию среды. |
