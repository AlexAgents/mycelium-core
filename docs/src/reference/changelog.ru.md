# Changelog

Все значимые изменения MYCELIUM CORE документируются в этом файле.
Формат основан на [Keep a Changelog](https://keepachangelog.com/en/1.1.0/).
Проект придерживается [Semantic Versioning](https://semver.org/spec/v2.0.0.html).

## [1.0.1] — 2026-05-20

Технический релиз. Критические исправления ошибок и улучшения качества
кода, выявленные в ходе code review перед защитой. Обратная
совместимость не нарушена.

### Добавлено

#### Core / Backend

- DTO `VoterStatus` в `src/core/voter_status.py` — типизированный
  снимок состояния избирателя (whitelist, has_voted, balance, stage)
  для потребления UI.
- `AppController.get_voter_status()` — метод единственного вызова,
  агрегирующий несколько RPC-запросов, возвращает `VoterStatus`.
- `VoterStatusWorker` в `src/ui/workers/voter_status_worker.py` —
  фоновый worker для неблокирующего обновления статуса избирателя.
- 12 новых i18n-ключей `err.parser.*` для всех разобранных ошибок RPC
  и контракта (недостаточно средств, конфликт nonce, таймаут,
  NotWhitelisted, AlreadyVoted, CandidateNotFound, InvalidStage,
  Unauthorized, общий revert контракта), а также 3 ключа действий для
  диалогов с призывом к действию.

### Изменено

- `ErrorParser.parse()` теперь возвращает i18n-ключи (`message_key`,
  `action_key`) вместо захардкоженных строк. UI самостоятельно вызывает
  `t(key)`, сохраняя строгое разделение слоёв.
- Структура payload `AppController.parse_rpc_error()` изменена:
  `{message_key, raw_message, action_key, action_id}` вместо
  `{message, action, action_id}`.
- `VoteTab._check_voter_status` заменён на дебаунсированный асинхронный
  запрос через `VoterStatusWorker` — UI больше не блокируется на
  RPC-вызовах при вводе приватного ключа.
- `VoteTab._sec06_shown` перенесён с уровня модуля в атрибут
  экземпляра, корректно сбрасывается в `reset_ui()`.
- Русский перевод `admin.dialog.cannot_start.no_whitelist`: название
  кнопки внутри сообщения теперь совпадает с реальной подписью кнопки
  в русском UI.
- Тестовые фикстуры обновлены: используются корректные 40-символьные
  адреса Ethereum вместо заглушек вида `0x1`, `0xabc`.

### Исправлено

- `GethManager._force_kill_pid` был определён дважды в одном классе;
  второе определение молча перекрывало первое и теряло флаг `/T`
  (завершение дочерних процессов на Windows). Дубликат удалён.
- `Account.enable_unaudited_hdwallet_features()` вызывался при импорте
  модуля в `crypto.py`, несмотря на неиспользование HD-кошельков.
  Удалён во избежание изменения глобального состояния `eth_account`.
- `MainWindow.closeEvent` вызывал `geth.stop()` дважды (один раз через
  блок finally в `controller.shutdown()`, второй раз напрямую). Приводило
  к двойным вызовам `taskkill` и засорению логов. Прямой вызов удалён.
- Зависание UI (~50–200 мс) при каждом нажатии клавиши в поле ввода
  приватного ключа на вкладке Vote, вызванное тремя синхронными
  RPC-вызовами в UI-потоке. Исправлено переносом всего запроса статуса
  в `VoterStatusWorker`.

### Удалено

- Глобальная переменная `_sec06_shown` уровня модуля в `vote_tab.py`.
- Дублирующееся определение метода `_force_kill_pid` в
  `geth_manager.py`.
- Вызов `Account.enable_unaudited_hdwallet_features()` из `crypto.py`.
- Дублирующийся вызов `self.controller.geth.stop()` из
  `MainWindow.closeEvent`.

### Безопасность

- Все сообщения об ошибках, показываемые пользователю, теперь корректно
  локализованы — исключается проникновение английского текста ошибок
  в русский UI.

### Тесты

- **180 passed** (100.86s) — полный прогон включает интеграционные
  тесты с реальным узлом Geth.
- `test_integration.py` — 12 тестов полного цикла голосования (01–12).
- `pytest.ini`: `addopts = --strict-markers` — интеграционные тесты
  запускаются только с явным флагом `-m integration`; по умолчанию
  исключены.

## [1.0.0] — 2026-05-10

Финальный релиз. Все архитектурные, функциональные и UX-требования
исходного ТЗ выполнены. Готов к демонстрации и аудиту.

### Добавлено

#### Core / Backend

- `AppController.precheck_vote()` — полная предголосовая валидация на
  уровне доменного слоя.
- `AppController.build_full_report()` — единственный источник истины для
  экспорта аудита.
- `AppController.fund_from_dev()` — асинхронный перевод с dev-аккаунта
  Geth.
- `AppController.check_start_voting_ready()` — предварительная проверка
  перед запуском голосования.
- `AppController.get_chain_stats()` — статистика использования диска и
  сессии.
- `AppController.reset_blockchain_data()` — полный сброс с повторными
  попытками на Windows.
- `AppController.get_client_version()`, `is_contract_deployed()`,
  `get_contract_address()`, `is_dev_mode()` — фасадные геттеры.
- `ErrorParser` — выделен из AppController в `src/core/error_parser.py`.
- `PrecheckResult` / `PrecheckStatus` — типизированные объекты результата
  в `src/core/precheck.py`.
- `GethManager._taskkill_all_geth()` — защитная сеть от зомби-процессов.
- `GethManager._is_port_in_use()` — предварительная проверка порта.
- Флаг `GethManager._shutting_down` — предотвращает ложные
  crash-callbacks.

#### UI / UX

- Диалог «О программе» с логотипом, версией, автором и ссылкой на
  GitHub.
- Кнопка «Сбросить данные блокчейна» в шапке с диалогом подтверждения.
- Предупреждение при запуске при большом размере папки chain-data
  (>500 МБ).
- Индикатор доступности с учётом стадии в режиме Audit.
- Раздел Voters: заблокирован до деплоя контракта.
- Раздел Candidates: заблокирован после регистрации или вне SETUP.
- Поле ввода admin key: только для чтения после деплоя.
- Предварительные проверки с учётом стадии перед vote/mass-vote.
- Кнопка «Fund from Dev» (только иконка) для управления балансом
  в dev-режиме.
- Очередь toast-уведомлений с дедупликацией.
- Вкладка Logs: живой фильтр поиска, флажок автопрокрутки,
  информационная панель (строки / размер / путь).
- Кнопки About / Theme / Exit — только иконки (более чистая шапка).

#### Архитектура

- Единственный источник истины для полного отчёта (`build_full_report`).
- Строгое разделение: UI никогда не импортирует `web3`, `eth_account`,
  `solcx`.
- Единый виджет `MessageDialog`, заменяющий разрозненные вызовы
  `QMessageBox`.
- Утилита `msgbox_helpers` для стандартных кнопок с переводом.
- Фикстура pytest `conftest.py` для чистых тестовых запусков.
- Logger автоматически архивирует предыдущий `session.log` при запуске.

#### Документация

- Architecture Decision Records (ADR-001 — ADR-007).
- Подробный README с инструкциями по настройке.
- Inline-документация для всех публичных методов.

### Изменено

- Geth запускается с `--dev.period 5` для стабильного запечатывания
  блоков.
- chain-data Geth очищается при каждом запуске.
- Отображение баланса переведено на научную нотацию для очень больших
  значений (`1.16e+59 ETH`).
- Заголовки `QGroupBox` переработаны.
- Улучшен контраст `StatusBadge` в светлой теме.
- Toasts: стековая очередь вместо параллельного отображения.
- Все захардкоженные строки перенесены в `i18n/ru.json` и `i18n/en.json`.
- Язык по умолчанию при первом запуске изменён на английский.

### Исправлено

- Ошибка сбора pytest (`ModuleNotFoundError: src`) через `pytest.ini`
  в корне и `conftest.py`.
- Диалог краша Geth, показывавшийся при намеренном завершении
  (добавлен флаг `_shutting_down`).
- Toast, отображавшийся поверх других приложений Windows.
- Дублирующиеся toasts при двойном переключении языка.
- Переполнение метки баланса администратора на узких окнах.
- Кнопки, зависящие от стадии, не обновлялись без ручного обновления.
- Geth manager: файловые блокировки на Windows после остановки.
- Вкладка Logs: автоперезагрузка при переключении вкладок.
- `session.log` не очищался между перезапусками приложения.
- Светлая тема: чёрный текст заменён тёмно-синими тонами.

### Удалено

- `audit_worker.py` (заменён встроенным `_StagedAuditWorker`).
- Мёртвая константа `_SENTINEL_PATTERNS` в logger.
- Неиспользуемый путь `SOLC_BIN`.
- Дублирующиеся проверки аудита в `_run_checks` для полного режима.
- Секция `[dev]` из значений по умолчанию `app.cfg`.
- `parse_rpc_error()` — статический метод удалён и заменён методом
  экземпляра.

### Безопасность

- `session.log` автоматически редактирует строки, соответствующие
  паттерну приватного ключа.
- Экспорт избирателей требует явного подтверждения пользователя.
- Предупреждение SEC-06 показывается при первом вводе приватного ключа
  за сессию.
- Все административные действия проверяются на соответствие владельцу
  контракта в `_check_owner_actions`.

## [0.3.0] — 2024-03-14

### Добавлено

- Mass Vote с предварительной фильтрацией (проверка whitelist и баланса).
- Улучшенная обработка ошибок: неблокирующий UI, предложения повтора.
- Индикатор прогресса в статусной строке с градиентной заливкой.
- Интернационализация (`ru.json`, `en.json`) с переключением языка
  в рантайме.
- Проверка аудита «только для владельца» для административных действий.
- События `VotingStarted` и `VotingFinished` в смарт-контракте.
- Toast-уведомления (снизу справа, окрашены по типу).

### Изменено

- Аудит рефакторен в поэтапные проверки (pre-vote, live, final, full).
- Светлая тема: покрыты все виджеты, отсутствующие стили добавлены.
- Отображение баланса: формат ETH с адаптивной точностью.

### Исправлено

- Receipt QR сбрасывался при запуске mass vote.
- New Session: полная очистка таблиц UI.
- Вкладка Logs: цвета иконок приведены в соответствие с темой.
- Победитель отображается только после стадии FINISHED.
- Раскладка Voters: распределена по всей строке.

## [0.2.0] — 2024-03-10

### Добавлено

- Кнопка Register candidates: активна только при наличии ≥2 кандидатов.
- Кнопка Fund Voters для распределения газа.
- Функция Mass Vote (JSON-файл или текущая сессия).
- Индикатор режима Geth в шапке.
- Кнопка Exit с диалогом подтверждения.
- Кнопка New Session: архивирование логов, сброс состояния, опциональный
  автодеплой.
- Вкладка Logs как отдельный раздел UI.
- Поэтапный аудит (pre-vote / during / post-vote).
- Импорт JSON с избирателями через drag-and-drop.
- Автозагрузка кандидатов на вкладке VoteTab после регистрации.
- Подтверждение закрытия при наличии активных worker-задач.
- Автозаполнение `DEV_ADMIN_KEY` в dev-режиме.

### Исправлено

- Логика блокировки кнопок в разделе Voters после добавления в
  whitelist.
- Множественные архитектурные правки согласно разделу 6 ТЗ.

## [0.1.0] — 2024-03-01

Первоначальная реализация на основе ТЗ v1.0.

- Смарт-контракт `VotingCore.sol`.
- Сервисный слой (`VotingService`, `AuditService`, `NonceManager`).

- Три основные вкладки: Admin, Vote, Audit.
