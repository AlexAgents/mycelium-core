# Слои системы

Этот документ содержит подробное описание обязанностей, ограничений и компонентов каждого архитектурного слоя в **MYCELIUM CORE**.

---

## 1. Слой представления (UI)

- **Расположение:** `src/ui/`
- **Основные компоненты:** `MainWindow`, `AdminTab`, `VoteTab`, `AuditTab`, `LogsTab` и кастомные виджеты (`Toast`, `StatusBadge`).
- **Обязанности:** Отрисовка графического интерфейса, захват пользовательского ввода, базовая валидация и отправка задач на прикладной слой.
- **Ограничения:**
    - **Никогда** не должен импортировать `web3`, `eth_account` или `solcx`.
    - Должен взаимодействовать с бэкендом строго через фасад `AppController`.
    - Все длительные операции должны запускаться в отдельных потоках с использованием `ThreadRunner` и тонких подклассов `BaseWorker`.

## 2. Прикладной слой (Фасад)

- **Расположение:** `src/core/app_controller.py`
- **Основные компоненты:** Класс `AppController`.
- **Обязанности:** Роль посредника (фасада) между UI и доменными сервисами, поддержка контекста сессии (`SessionContext`) и координация высокоуровневых рабочих процессов приложения (таких как инициализация, создание новых сессий и безопасный останов).
- **Ограничения:**
    - Не должен содержать низкоуровневой логики построения транзакций блокчейна.
    - Должен предоставлять чистые, понятные для UI методы, изолируя интерфейс от внутренних доменных моделей.

## 3. Доменный слой / Сервисы

- **Расположение:** `src/core/`
- **Основные компоненты:** `VotingService`, `AuditService`, `ErrorParser`, `PrecheckResult`.
- **Обязанности:** Инкапсуляция основных бизнес-правил процесса голосования.
    - `VotingService` управляет взаимодействием со смарт-контрактом, подписью транзакций и деплоем.
    - `AuditService` анализирует события блокчейна для составления независимой статистики и выполнения криптографических проверок.
    - `ErrorParser` преобразует сырые сетевые ошибки RPC в понятные человеку сообщения.

## 4. Слой инфраструктуры

- **Расположение:** `src/core/` и `src/utils/`
- **Основные компоненты:** `Web3Provider`, `GethManager`, `NonceManager`, `CompilerService`.
- **Обязанности:** Взаимодействие с внешней средой, операционной системой и сетью Ethereum. Он запускает/останавливает Geth, компилирует код Solidity на лету, предоставляет единое подключение Web3 и управляет потокобезопасными nonce.
