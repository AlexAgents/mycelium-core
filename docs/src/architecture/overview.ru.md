# Обзор архитектуры

Этот документ содержит высокоуровневое описание архитектуры системы **MYCELIUM CORE**. Приложение построено с использованием строгого шаблона слоистой архитектуры для обеспечения поддерживаемости, тестируемости и разделения обязанностей.

---

## Высокоуровневая блок-схема

Поток взаимодействия между компонентами системы следует модели
однонаправленной зависимости от слоя представления к смарт-контракту.
Детальный вид на уровне компонентов — в
[Диаграмме компонентов](../diagrams/architecture/component-diagram.md).

```text
┌─────────────────────────────────────────────────────────┐
│                 СЛОЙ ПРЕДСТАВЛЕНИЯ (UI)            │
│                 src/ui/ (Виджеты PyQt6)            │
└──────────────────────────────┬──────────────────────────┘
                             │
                             ▼
┌─────────────────────────────────────────────────────────┐
│               ПРИКЛАДНОЙ СЛОЙ (Фасад)              │
│               src/core/app_controller.py           │
└──────────────────────────────┬──────────────────────────┘
                             │
                             ▼
┌─────────────────────────────────────────────────────────┐
│                ДОМЕННЫЙ СЛОЙ / СЕРВИСЫ             │
│             src/core/ (Voting/Audit Services)      │
└──────────────────────────────┬──────────────────────────┘
                             │
                             ▼
┌─────────────────────────────────────────────────────────┐
│                СЛОЙ ИНФРАСТРУКТУРЫ                 │
│        src/core/ (Web3Provider / GethManager)      │
└──────────────────────────────┬──────────────────────────┘
                             │
                             ▼
┌─────────────────────────────────────────────────────────┐
│                    СМАРТ-КОНТРАКТ                  │
│                 contracts/VotingCore.sol           │
└─────────────────────────────────────────────────────────┘
```

## Архитектурные принципы

1. **Однонаправленная зависимость:** Верхние слои могут ссылаться на
   нижние, но нижние слои никогда не должны импортировать компоненты
   из верхних.
2. **Изоляция UI от Web3:** Слой представления полностью отделён от
   классов Web3 и Ethereum. Он взаимодействует исключительно с фасадом
   `AppController`.
3. **Единый источник истины:** Смарт-контракт, развёрнутый в локальной
   приватной сети, служит абсолютным источником истины для всех состояний
   выборов, кандидатов, whitelist и голосов.
4. **Изоляция потоков:** Длительные транзакции блокчейна и операции
   управления узлом делегируются асинхронным воркерам в отдельных
   потоках, что предотвращает зависание интерфейса.
