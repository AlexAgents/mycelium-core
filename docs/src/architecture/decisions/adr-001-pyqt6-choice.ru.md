# ADR-001: Выбор PyQt6 вместо PySide6

## Статус
Принято

## Дата
2026-05-20

## Контекст
Оригинальное техническое задание рекомендовало PySide6 в качестве основной библиотеки интерфейса. Однако на этапе реализации была выбрана PyQt6 из-за совместимости с внешними библиотеками и более стабильной сборки через PyInstaller в целевой среде Windows.

## Решение
Мы решили использовать PyQt6 (>=6.7.0) для слоя представления (UI).

## Последствия

### Положительные

- Стабильная работа окон в Windows 10/11.
- Лучшая совместимость со сборщиком иконок `qtawesome`.
- Бесшовная сборка через PyInstaller без сложной настройки хуков импорта.

### Отрицательные

- Отклонение от PySide6 (официальной библиотеки Qt Company).
- Потребуются небольшие правки импортов при переносе на PySide6 в будущем.
