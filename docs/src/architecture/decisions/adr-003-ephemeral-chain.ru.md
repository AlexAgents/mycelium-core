# ADR-003: Очистка данных блокчейна при старте

## Статус
Принято

## Дата
2026-05-20

## Контекст
Geth `--dev` сохраняет заголовки блоков на диске, но сбрасывает состояние аккаунтов из памяти при выключении. Это приводит к рассинхронизации базы данных при следующем запуске и ошибкам вида `missing trie node` и `historical state not available`.

## Решение
Мы решили выполнять полную очистку папки `chain-data/active/` при каждом запуске приложения.

## Последствия

### Положительные

- Гарантированно чистое состояние при старте: Geth никогда не выдаёт ошибок повреждения базы данных.
- Стабильный майнинг и отправка транзакций.
- Отсутствие разрастания диска (размер активной цепи всегда минимален).

### Отрицательные

- История блоков не сохраняется между перезапусками приложения.
- Адреса развёрнутых контрактов должны пересоздаваться при каждом запуске.
