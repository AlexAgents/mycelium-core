# ADR-007: Выделение ErrorParser и Precheck из AppController

## Статус
Принято

## Дата
2026-05-20

## Контекст
По мере добавления проверок перед голосованием и обработки ошибок RPC, класс `AppController` разрастался, нарушая принцип единственной ответственности.

## Решение
Мы решили выделить логику парсинга и валидации в отдельные доменные модули: `ErrorParser` (для перевода ошибок) и `Precheck` (содержащий `PrecheckResult` и enum `PrecheckStatus`). `AppController` делегирует задачи этим сервисам.

## Последствия

### Положительные

- Сохраняет `AppController` компактным и поддерживаемым.
- Изолированное тестирование сценариев `ErrorParser`.
- Строго типизированные объекты результатов `PrecheckResult` вместо словарей.

### Отрицательные

- Добавление двух новых файлов в кодовую базу.
