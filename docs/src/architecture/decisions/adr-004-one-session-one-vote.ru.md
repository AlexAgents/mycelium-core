# ADR-004: Принцип "Одна сессия — Одно голосование"

## Статус
Принято

## Дата
2026-05-20

## Контекст
Смарт-контракт, разработанный для выборов, должен быть безопасным и одноразовым. Повторное использование контракта или сброс его стадий назад несёт риски для безопасности и нарушает целостность аудита.

## Решение
Мы решили применить строгое правило "Одна сессия = Один контракт". Стадии `VotingCore.sol` движутся строго вперёд: `SETUP → ACTIVE → FINISHED`. Сброс стадий невозможен на уровне смарт-контракта. Для новых выборов требуется деплой нового контракта.

## Последствия

### Положительные

- Гарантирует честность выборов: завершённые голоса никогда не могут быть изменены или сброшены.
- Простая модель аудита: все события в рамках контракта сессии относятся к одним выборам.
- Идеально сочетается с эфемерной природой блокчейна Geth dev-mode.

### Отрицательные

- Затраты на газ для деплоя должны оплачиваться при каждых новых выборах.
