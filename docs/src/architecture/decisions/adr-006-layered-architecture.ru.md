# ADR-006: Слоистая архитектура

## Статус
Принято

## Дата
2026-05-20

## Контекст
Смешивание логики транзакций блокчейна напрямую в виджетах Qt создаёт спагетти-код, нарушает разделение обязанностей и делает тестирование невозможным.

## Решение
Мы решили применить строгую слоистую архитектуру: `UI -> AppController (Фасад) -> Сервисы -> Инфраструктура -> Смарт-контракт`. Слою UI запрещено импортировать классы Web3 или контрактов. Все операции должны проходить через фасад `AppController`.

## Последствия

### Положительные

- Чистый код: файлы UI обрабатывают только макеты, стили и базовую валидацию.
- Тестируемость: доменные сервисы (`AuditService`, `ErrorParser`) могут быть полностью протестированы без PyQt.
- Простая модификация UI: стили или виджеты можно менять, не затрагивая доменную логику.

### Отрицательные

- Небольшой шаблонный код (boilerplate): добавление функции требует проброса вызовов через контроллер.
