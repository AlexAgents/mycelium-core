# ADR-002: Использование Geth Dev Mode вместо приватной сети PoA

## Статус
Принято

## Дата
2026-05-20

## Контекст
Для песочницы нам нужен локальный блокчейн с быстрым подтверждением, нулевой сложностью настройки и предоплаченными аккаунтами для финансирования кошельков администратора и избирателей.

## Решение
Мы решили использовать Geth в режиме `--dev` с автоматической генерацией
блоков каждые 5 секунд (`--dev.period 5`).

Параметр `--dev.period 5` означает, что новый блок запечатывается каждые
5 секунд даже при отсутствии транзакций. Это обеспечивает предсказуемое
движение цепи для проверок аудита: SEC-03 (Stage Enforcement) опирается
на монотонные номера блоков между событиями `VotingStarted` и
`VotingFinished`. Меньшие значения (`1`) тестировались, но приводили к
избыточной нагрузке на CPU и шумным логам при простаивающем UI.

## Последствия

### Положительные

- Нулевая настройка: Geth запускается мгновенно без файла генезиса.
- Предоплаченный аккаунт разработчика доступен сразу как `eth.coinbase`.
- Предсказуемое запечатывание блоков каждые 5 секунд упрощает проверку аудита.

### Отрицательные

- Эфемерное состояние: Geth `--dev` хранит состояние в оперативной памяти; данные сбрасываются при перезапуске.
- Не является полной симуляцией многоузловой сети Proof-of-Authority (PoA).
