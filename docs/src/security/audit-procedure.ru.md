# Стандартная операционная процедура (SOP): Аудит выборов
Пошаговое руководство для аудиторов и исследователей по верификации завершённой сессии голосования с использованием MYCELIUM CORE.

---

## 1. Предварительные условия

- Стадия голосования строго `FINISHED` (Администратор нажал "Завершить голосование").
- Приложение запущено, статус RPC — `CONNECTED`.
- Адрес смарт-контракта отображается в нижней части (footer) окна приложения.

---

## 2. Процедура выполнения
### Шаг 2.1: Инициализация модуля аудита
Перейдите на вкладку **Аудит** в главном окне приложения.

### Шаг 2.2: Выбор области выполнения
Выберите **Полный аудит (Full Audit)** в селекторе режимов. Этот режим координирует все проверки с SEC-01 по SEC-06, а также структурные валидации pre-vote.

### Шаг 2.3: Запуск верификации
Нажмите **Запустить аудит**. Система асинхронно загрузит все соответствующие события блокчейна и вычислит криптографические инварианты. Дождитесь завершения индикатора прогресса.

### Шаг 2.4: Оценка матрицы безопасности
Изучите таблицу проверок безопасности. Интерпретируйте флаги следующим образом:
| Флаг статуса | Значение | Требуемое действие |
|---|---|---|
| `PASSED` (зелёный) | Криптографический инвариант соблюдён | Действий не требуется |
| `FAILED` (красный) | Критическое нарушение инварианта | Немедленно изучите столбец "Детали" |
| `WARNING` (жёлтый) | Невозможно однозначно верифицировать (напр., нет метаданных блока) | Проверьте доступность данных цепи |
| `SKIPPED` (серый) | Неприменимо в выбранном режиме | Переключитесь в режим "Полный аудит" |

### Шаг 2.5: Проверка согласованности подсчётов
В разделе "Результаты" убедитесь, что:

- Количество голосов за кандидатов соответствует ожиданиям и off-chain прогнозам.
- Исход "Победитель / Ничья" корректно выведен из результатов подсчета.
- Общее количество голосов в точности совпадает с числом проголосовавших избирателей в логах событий.

### Шаг 2.6: Создание неизменяемой записи (Артефакта)
Воспользуйтесь одним из вариантов экспорта для создания неизменяемого следа аудита:

- **Копировать отчёт** — JSON в буфер обмена (для быстрой передачи).
- **JSON** — генерирует полный артефакт в файл `.json`.
- **CSV** — генерирует структурированный отчёт с разделами `[Results]`, `[Winner]`, `[Audit]` для анализа в таблицах.

### Шаг 2.7: Архивирование сессии
Нажмите **Новая сессия** в шапке приложения. Текущий `session.log` будет безопасно заархивирован в `data/logs/archive/<session_id>/` для будущих расследований.

---

## 3. Финальный чеклист аудитора

- [ ] Все 6 проверок SEC отображают статус `PASSED`.
- [ ] Количество голосов в таблице "Результаты" математически совпадает с количеством событий в подробностях SEC-06.
- [ ] Имя и партия победителя логически корректны на основе результатов подсчета.
- [ ] Все статусы `WARNING` получили объяснение.
- [ ] Экспортированный JSON-отчёт безопасно сохранён в архивном хранилище организации.
- [ ] Архивированный журнал сессии проверен на наличие неожиданных записей с уровнем `ERROR`.
