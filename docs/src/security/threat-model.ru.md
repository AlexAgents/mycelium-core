# Модель угроз (STRIDE)

## Заметка аналитика: Локальное выполнение с нулевым доверием
В данном документе анализируются угрозы безопасности системы **MYCELIUM CORE** с использованием методологии STRIDE. Поскольку это автономное десктопное приложение, поверхность атак значительно отличается от стандартных веб-приложений. Здесь нет внешнего сетевого воздействия, публичного мемпула и многопользовательского доступа. Поэтому модель угроз предполагает **локальную среду с нулевым доверием (Zero-Trust Local Environment)**, где основными векторами атак являются манипуляции с локальными файлами и кража учетных данных.

```text

| Машина разработчика           |

|  +----------+      JSON-RPC   |
|  | MYCELIUM |<--------------> |
|  | CORE     |     (loopback)  |
|  +----------+      +------+   |
|       |            | Geth |   |
|       v            +------+   |
|  +-----------+                |
|  | Файловая  |                |
|  | система   |                |
|  | (журналы, |                |
|  | цепочка,  |                |
|  | ключи)    |                |
|  +-----------+                |
+-------------------------------+
```

## Анализ по методологии STRIDE

### S — Spoofing (Подмена идентичности)
| Угроза | Последствие | Компенсирующая мера | Остаточный риск |
|---|---|---|---|
| Злоумышленник получает приватный ключ избирателя | Голос подаётся посторонним лицом | Предупреждение SEC-06 при первом вводе ключа; ключи по умолчанию не сохраняются; `_SecretFilter` исключает ключи из журналов | **Принят** — неотъемлемое ограничение аутентификации на основе ключей в отсутствие аппаратных кошельков |
| Злоумышленник получает приватный ключ администратора | Полный контроль над контрактом | Ключ по умолчанию не сохраняется; режим разработки явно помечен; поле ключа только для чтения после деплоя | Средний — компромисс между удобством dev-режима и безопасностью |

### T — Tampering (Подделка данных)
| Угроза | Последствие | Компенсирующая мера | Остаточный риск |
|---|---|---|---|
| Изменение `VotingCore.sol` перед компиляцией | Модифицированная логика контракта | Валидация хэша ABI через `validate_abi_hash()`; исходный код под версионным контролем | Низкий |
| Подделка `session.log` | Фальсификация журнала аудита | Журнал доступен только для дозаписи; архивируется при новой сессии; `_SecretFilter` предотвращает утечку ключей | Низкий — криптографическая подпись не применяется |
| Изменение экспортированных JSON-результатов | Фальсификация отчёта аудита | `build_full_report()` читает данные из событий цепочки, а не из файлов | Низкий |

### R — Repudiation (Отказ от авторства)
| Угроза | Последствие | Компенсирующая мера | Остаточный риск |
|---|---|---|---|
| Избиратель отрицает факт голосования | Невозможность урегулирования споров | QR-квитанция с хэшем транзакции обеспечивает доказательство; событие `VoteCast` неизменяемо | Низкий |
| Администратор отрицает факт смены стадии | Невозможность установления ответственности | Событие `StageChanged` в блокчейне; `_check_owner_actions` (SEC-05) верифицирует отправителей | Низкий |

### I — Information Disclosure (Утечка информации)
| Угроза | Последствие | Компенсирующая мера | Остаточный риск |
|---|---|---|---|
| Запись приватных ключей в session.log | Компрометация ключей | Маскировка regex в `_SecretFilter`; блокировка слов (`private_key`, `mnemonic`, `secret`) | Низкий |
| Экспорт JSON-файла избирателей с ключами | Компрометация ключей при передаче файла | Предупреждение в `ConfirmDialog` перед экспортом; файл сохраняется только по явному действию | Средний — ответственность пользователя |
| Ключ администратора в файле `.env` | Компрометация ключа при попадании в git | `.env` включён в `.gitignore`; бейдж режима разработки в UI | Средний |

### D — Denial of Service (Отказ в обслуживании)
| Угроза | Последствие | Компенсирующая мера | Остаточный риск |
|---|---|---|---|
| Порт 8545 занят другим процессом | Приложение не может запуститься | Проверка `_is_port_in_use()` и подстраховка через `taskkill` | Низкий |
| Аварийное завершение процесса Geth | Транзакции могут завершиться ошибкой | `MonitorThread` обнаруживает падение; `_crash_signal` уведомляет UI; флаг `_shutting_down` предотвращает ложные срабатывания | Низкий |
| Большой объём данных цепочки на диске | Замедление запуска | `StartupWarnDialog` при >500 МБ; `reset_blockchain_data()` с повторными попытками | Низкий |

### E — Elevation of Privilege (Повышение привилегий)
| Угроза | Последствие | Компенсирующая мера | Остаточный риск |
|---|---|---|---|
| Вызов админ-функций не владельцем | Несанкционированная смена стадий | Модификатор `onlyOwner` в `VotingCore.sol`; проверка аудита SEC-05 | Низкий — исполнение на уровне блокчейна |
| Подача голоса адресом не из whitelist | Учёт недействительного голоса | `require(whitelist[msg.sender])` в контракте; проверка аудита SEC-02 | Низкий — исполнение на уровне блокчейна |
| Повторное голосование | Двойной голос | `require(!hasVoted[msg.sender])` в контракте; проверка аудита SEC-01 | Низкий — исполнение на уровне блокчейна |

## Угрозы вне области применения
Перечисленные ниже угрозы явно **не рассматриваются** в MYCELIUM CORE в соответствии с SRS (Раздел 3.2):

- **Сетевые атаки** (MITM, подмена DNS) — неприменимы, весь трафик проходит через loopback.
- **Атаки по сторонним каналам** на деривацию ключей — выходят за рамки песочницы.
- **Атаки Сибиллы** — whitelist формируется администратором, открытая регистрация не предусмотрена.
- **Принуждение избирателей / подкуп голосов** — требует криптографической анонимности (вне области применения).
- **Формальная верификация смарт-контракта** — не выполнялась; контракт намеренно упрощён.
- **Атаки на консенсус в мультиузловой сети** (51%) — единственный узел в режиме разработки, неприменимо.
