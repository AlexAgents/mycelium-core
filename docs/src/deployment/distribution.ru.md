# Дистрибуция и релизы

Порядок публикации и верификации релизов **MYCELIUM CORE**.

---

## Структура релиза

Релиз включает в себя:

```text
dist/
├── MYCELIUM_CORE.exe    # Автономный исполняемый файл
└── checksums.txt        # SHA-256 хэши
```

А также внешний файл `bin/geth.exe`, распространяемый вместе с релизом.

---

## Создание релиза

1. Собрать исполняемый файл:
   ```bash
   python scripts/builder.py --build
   ```

2. Проверить контрольные суммы:
   ```bash
   python scripts/builder.py --checksums
   ```

3. Создать GitHub Release с тегом `v1.0.x`.
4. Прикрепить `MYCELIUM_CORE.exe` и `checksums.txt`.

---

## Верификация

Получатели проверяют целостность файлов:

```powershell
Get-FileHash "MYCELIUM_CORE.exe" -Algorithm SHA256 | Format-List
```

```bash
sha256sum -c checksums.txt
```

---

## Важные замечания

- `bin/geth.exe` **не** включается внутрь EXE-файла. Распространяйте его в папке `bin/` вместе с исполняемым файлом.
- EXE-файл включает: QSS-темы, JSON-словари i18n, исходные файлы контракта и иконки приложения.
- Рабочие данные (`data/`) создаются автоматически при первом запуске.
