# Сборка исполняемого файла (PyInstaller)

Это руководство описывает, как скомпилировать исходный Python-код **MYCELIUM CORE** в один автономный исполняемый файл с помощью автоматического скрипта сборки.

---

## Скрипт автоматизации сборки

Проект содержит комплексный скрипт сборки в `scripts/builder.py`.
Скрипт управляет скрытыми импортами, генерацией иконок, расчётом
контрольных сумм SHA-256 и предоставляет локализованное интерактивное
меню (EN / RU).

### Как собрать

1. Откройте терминал в корневой папке проекта.
2. Запустите скрипт сборки:
   ```bash
   python scripts/builder.py
   ```

3. Выберите пункт **[1] Собрать EXE** в интерактивном меню.

### Неинтерактивный режим
Для CI или скриптовых сборок передавайте флаги напрямую:

```bash
python scripts/builder.py --build         # сборка без меню
python scripts/builder.py --clean-all     # очистка build/ + dist/
python scripts/builder.py --checksums     # вывод SHA-256 из dist/
python scripts/builder.py --lang ru       # принудительно русский вывод
```

### Детали процесса
Скрипт запускает `PyInstaller` со следующими параметрами:

- `--onefile` — Упаковывает все модули Python и зависимости в один исполняемый файл.
- `--windowed` — Собирает приложение без консольного окна (только для Windows).
- `--add-data` — Упаковывает темы QSS, JSON-словари локализации и файлы контрактов прямо внутрь бинарного файла.

> **Важное примечание:** Внешний процесс `bin/geth.exe` **не** упаковывается внутрь EXE во избежание избыточного размера файла. Размещайте `geth.exe` в папке `bin/` рядом с вашим скомпилированным файлом.
