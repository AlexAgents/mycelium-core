# Устранение неполадок

Решения типичных проблем, возникающих при работе с **MYCELIUM CORE**.

---

## Geth и сеть

### Порт 8545 уже занят
```bash
taskkill /F /IM geth.exe /T
```
Перезапустите приложение. Если порт по-прежнему занят, его использует другой процесс — измените значение `RPC_PORT` в файле `.env`.

### Ошибка «Geth not found»
Убедитесь, что файл `bin/geth.exe` (Windows) или `bin/geth` (Linux/macOS) существует и имеет права на выполнение.

### Geth запускается, но RPC так и не подключается
Проверьте файл `data/chain-data/active/geth.log` на наличие сообщений об ошибках. Наиболее распространённая причина: повреждённые данные цепочки. Решение: нажмите **Reset Data** в заголовке окна или вручную удалите папку `data/chain-data/active/`.

### Geth аварийно завершает работу сразу после запуска
Каталог данных цепочки мог быть повреждён в результате предыдущего некорректного завершения работы. Удалите папку `data/chain-data/active/` и перезапустите приложение.

### Блоки не добываются
Geth в режиме `--dev` с параметром `--dev.period 5` добывает блок каждые 5 секунд вне зависимости от наличия ожидающих транзакций. Если прирост блоков прекратился, Geth, по всей видимости, аварийно завершил работу — проверьте индикатор статуса в нижней части окна.

## Контракт и транзакции

### Ошибка «Contract not compiled» при развёртывании
Приложение компилирует `contracts/VotingCore.sol` в режиме реального времени. Убедитесь, что файл существует и не повреждён. Компилятор Solidity загружается автоматически через `py-solc-x` при первом использовании — убедитесь в наличии доступа к интернету.

### «Insufficient funds for gas»
На аккаунте администратора или избирателя отсутствует ETH. Для администратора: нажмите **Fund from Dev**. Для избирателей: нажмите **Fund** в разделе Voters.

### Ошибка «Nonce too low»
Счётчик nonce рассинхронизировался. Приложение выполняет повторные попытки автоматически (до 3 раз с экспоненциальной выдержкой). Если проблема сохраняется, создайте **новую сессию**.

### Кнопка Deploy остаётся неактивной
Кнопка Deploy блокируется после успешного развёртывания во избежание случайного повторного деплоя. Для повторного развёртывания создайте **новую сессию**.

### Истечение таймаута подтверждения транзакции
Транзакция была отправлена в сеть, однако не была включена в блок в течение 120 секунд. Проверьте, работает ли Geth. Транзакция может по-прежнему находиться в ожидании в мемпуле.

## UI и приложение

### Окно приложения не появляется
Запустите приложение из терминала для просмотра вывода ошибок: `python main.py`. Типичные причины: отсутствие PyQt6, отсутствие зависимостей, несоответствие версии Python.

### Язык меняется не полностью
Некоторые элементы интерфейса требуют перезапуска приложения для полного применения перевода. Об этом сообщается во всплывающем уведомлении при переключении языка.

### Всплывающие уведомления появляются позади других окон
Исправлено в v1.0.0. Уведомления отображаются как дочерние элементы центрального виджета. Если проблема сохраняется, убедитесь, что используется последняя версия приложения.

### На вкладке Logs отображается «(log file not found)»
Файл журнала создаётся при запуске приложения. Если файл отсутствует, приложение, возможно, не запустилось корректно. Проверьте вывод в терминале.

### Переключение темы не обновляет индикаторы статуса
Исправлено в v1.0.0. Все виджеты `StatusBadge` вызывают `refresh_theme()` при смене темы. Если индикаторы отображаются некорректно, нажмите кнопку переключения темы дважды.

## Тесты

### `ModuleNotFoundError: src` при запуске pytest
Убедитесь, что запуск выполняется из корневой директории проекта при наличии файла `pytest.ini`. Этот файл задаёт `pythonpath = .`, добавляя корневую директорию в `sys.path`.

```bash
cd mycelium-core
python -m pytest -v
```

### Загрузка компилятора Solidity завершается ошибкой во время тестирования
`test_compile_contract` выполняет реальную компиляцию. Убедитесь в наличии доступа к интернету при первом запуске. При последующих запусках используется кэшированный бинарный файл solc.

## Файловая система

### «chain-data files are locked» при сбросе данных
Windows может удерживать блокировки файлов в течение 1–2 секунд после остановки Geth. Приложение выполняет до 8 повторных попыток с постепенным увеличением интервала. Если все попытки завершились неудачей, вручную завершите процесс `geth.exe` через Диспетчер задач и удалите папку `data/chain-data/active/`.

### Дисковое пространство постепенно заканчивается
Данные цепочки очищаются при каждом запуске, однако архивы сессий накапливаются в папке `data/logs/archive/`. Используйте функцию **Reset Data** с установленным флажком «Also delete archived session logs» для очистки архива.
