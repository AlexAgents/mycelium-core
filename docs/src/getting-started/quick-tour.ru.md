# Быстрый обзор

Двухминутный обзор всех четырёх вкладок **MYCELIUM CORE**.

---

## Структура окна

```text
+-- Хедер ---------------------------------------------+
| MYCELIUM CORE    [Новая сессия] [Сброс] [Язык] [...] |
+-- Tabs ----------------------------------------------+
| [ Admin ] [ Голосование ] [ Аудит ] [ Логи ]         |
+-- Tab content ---------------------------------------+
|                                                      |
|                                                      |
+-- Подвал --------------------------------------------+
| Стату | Прогресс | Стадия | Connected | Блок | ...   |
+------------------------------------------------------+
```

---

## Вкладка Admin

Для роли **Администратор**. Используется для подготовки голосования.

Разделы:

- **Контракт** — ввод приватного ключа администратора, баланс,
  **Fund from Dev**, **Развернуть VotingCore**, адрес контракта,
  отображение текущей стадии.

- **Кандидаты** — добавление (Имя, Партия, Адрес), генерация тестовых
  адресов, регистрация в сети.

- **Избиратели** — генерация тестовых избирателей, импорт / экспорт
  JSON, финансирование избирателей газом, добавление в whitelist.
  Поддерживается перетаскивание JSON.

- **Управление стадией** — начать / завершить голосование.

Кнопки зависят от стадии: действия с избирателями блокируются после
старта голосования, действия с кандидатами — после регистрации.

---

## Вкладка Голосование

Для роли **Избиратель**. Используется для подачи одиночного голоса
или массового голосования.

Разделы:

- **Аутентификация** — вставка приватного ключа или загрузка JSON.
  Поля статуса (В whitelist / Голосовал / Стадия / Баланс) обновляются
  через асинхронный воркер — UI не зависает при вводе.

- **Выбор кандидата** — загрузка кандидатов из сети, выбор одного.
- **Действие** — Проголосовать (одиночный голос).
- **Квитанция** — TX Hash, кнопка копирования, QR-квитанция (140x140 px).
- **Массовое голосование (Тестирование)** — Из JSON-файла или
  Из сессии, выполняет предварительно отфильтрованный пакет с логом
  прогресса.

---

## Вкладка Аудит

Для роли **Аудитор / Исследователь**. Проверяет целостность выборов
по on-chain данным.

Разделы:

- **Результаты** — победитель, голоса по кандидатам, кнопка обновления.
- **Проверки безопасности** — селектор режима (Full / Pre / Live /
  Final), кнопка Запустить аудит, таблица шести SEC-проверок со
  статусами.

- **Экспорт** — Копировать отчёт (JSON в буфер), Export JSON, Export CSV.

Доступность режимов аудита зависит от текущей стадии контракта.
Неприменимые режимы заблокированы в комбобоксе.

---

## Вкладка Логи

Для всех ролей. Читает `data/logs/active/session.log` в реальном
времени.

Возможности:

- **Обновить** — перечитать с диска.
- **Поиск** — фильтр по подстроке вживую.
- **Автопрокрутка** — чекбокс для tailing'а лога.
- **Копировать всё** / **Сохранить как** — буфер или файл.
- **Очистить** — очищает только отображение (файл **не** удаляется).

Нижняя инфо-строка: количество строк, размер файла, полный путь.

---

## Действия в хедере

- **Новая сессия** — архивирует текущую сессию, создаёт новую в том
  же экземпляре Geth (быстрый режим).

- **Сброс данных (ТЕСТ)** — глубокий сброс: остановка Geth, очистка
  chain-data, перезапуск. См. ADR-003 для контекста.

- **Комбобокс языка** — переключение EN / RU мгновенно (некоторые
  надписи требуют перезапуска приложения для полной локализации).

- **Кнопка темы** — переключение dark / light.
- **О программе** — версия, лицензия, ссылка на GitHub.
- **Выход** — подтверждение и корректное завершение.

---

## Индикаторы в подвале

- **Статус** — результат последней операции.
- **Прогресс** — градиентная полоса для длительных операций.
- **Стадия** — бейдж SETUP / ACTIVE / FINISHED.
- **Connected / Offline** — статус RPC.
- **Geth** — режим (DEV / CUSTOM).
- **Client** — строка версии Geth.
- **Блок** — текущий номер блока (обновляется каждые 2 секунды).
- **Контракт** — адрес развёрнутого контракта или "не развёрнут".
