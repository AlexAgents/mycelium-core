# Установка

Это руководство описывает установку **MYCELIUM CORE** на чистую машину
с Windows 10/11. Linux и macOS поддерживаются экспериментально.

---

## Системные требования

| Компонент | Минимум | Рекомендуется |
|---|---|---|
| ОС         | Windows 10 64-bit            | Windows 11 64-bit |
| Python     | 3.11                         | 3.11.x последний патч |
| ОЗУ        | 4 ГБ                         | 8 ГБ |
| Диск       | 500 МБ                       | 2 ГБ |
| Сеть       | не требуется (полностью локально) | — |

---

## Шаг 1. Клонирование репозитория

```bash
git clone <repository-url> mycelium-core
cd mycelium-core
```

---

## Шаг 2. Создание виртуального окружения

```bash
python -m venv venv
```

Активация:

=== "Windows"

    ```bash
    venv\Scripts\activate
    ```

=== "Linux / macOS"

    ```bash
    source venv/bin/activate
    ```

---

## Шаг 3. Установка Python-зависимостей

```bash
pip install -r requirements.txt
```

Ожидаемые пакеты: `PyQt6`, `web3`, `eth-account`, `py-solc-x`,
`qrcode`, `Pillow`, `qtawesome`, `python-dotenv`.

---

## Шаг 4. Размещение Geth

Скачайте [go-ethereum](https://geth.ethereum.org/downloads/) для вашей
ОС и поместите бинарник в папку `bin/`:

```text
bin/
├── geth.exe         # Windows
└── geth             # Linux / macOS
```

Приложение ищет Geth **только внутри `bin/`** и не зависит от
системной установки.

---

## Шаг 5. Настройка окружения

Скопируйте пример конфигурации:

```bash
cp .env.example .env
```

Минимальный `.env`:

```text
RPC_HOST=127.0.0.1
RPC_PORT=8545
DEV_MODE=true
DEV_ADMIN_KEY=0x...
```

`DEV_ADMIN_KEY` — удобство для тестирования: приложение автоматически
подставит этот ключ в поле Admin Key. **Не используйте этот режим в
продакшен-подобных сценариях.**

Для генерации тестового приватного ключа без запуска приложения:

```bash
python -c "from src.utils.crypto import generate_eth_keypair; \
           k,a = generate_eth_keypair(); \
           print('Key:', k); print('Address:', a)"
```

---

## Шаг 6. Проверка установки

```bash
python main.py
```

При первом запуске приложение:

1. Запускает локальный узел Geth из `bin/`.
2. Ожидает готовности RPC до 30 секунд.
3. Создаёт новую сессию голосования.
4. Открывает главное окно.

Если в подвале окна отображается статус **CONNECTED**, установка
выполнена успешно.

## Устранение неполадок

### Порт 8545 уже занят

При старте приложение пытается завершить зависшие процессы `geth.exe`.
Если порт всё ещё занят, выполните:

```bash
taskkill /F /IM geth.exe /T
```

Затем перезапустите приложение.

### Ошибка "Geth not found"

Проверьте, что бинарник существует по пути `bin/geth.exe` (Windows)
или `bin/geth` (Linux/macOS) и имеет права на выполнение.

### Не скачивается компилятор Solidity

`py-solc-x` скачивает компилятор при первом запуске. Убедитесь в
наличии интернета при первом деплое. Последующие запуски используют
кешированную версию.
