# Первый запуск

Это руководство описывает первый полный цикл голосования в
**MYCELIUM CORE** — от деплоя до аудита. Время выполнения: 5 минут.

---

## Что происходит при запуске

При запуске `python main.py`:

1. **Запускается Geth** (~2 секунды). Бейдж в подвале меняется с
   `OFFLINE` на `CONNECTED`.

2. **Создаётся новая сессия.** Генерируется UUID, открывается файл лога.
3. **Открывается главное окно** на вкладке Admin в стадии `SETUP`.

Если виден красный бейдж `GETH CRASHED`, см. раздел
[Устранение неполадок](installation.md).

---

## Выполните полный цикл голосования

### 1. Вкладка Admin — деплой контракта

1. Поле **Admin Private Key** автоматически заполняется в dev-режиме.
2. Нажмите **Fund from Dev** для перевода 100 ETH с dev-аккаунта на
   admin-адрес.

3. Нажмите **Deploy VotingCore**.

Дождитесь тоста `Контракт развёрнут`. Адрес контракта появится в подвале.

### 2. Добавление кандидатов

1. Заполните **Имя**, **Партия**, **Ethereum-адрес**. Кнопка
   **Сгенерировать** создаст тестовый адрес автоматически.

2. Нажмите **Добавить** для добавления кандидата в список.
3. Повторите минимум для 2 кандидатов (контракт требует `>= 2`).
4. Нажмите **Зарегистрировать в сети**.

Дождитесь тоста `N кандидатов зарегистрировано`. Поля кандидатов
заблокируются.

### 3. Генерация избирателей

1. Установите **Количество** в `5` (любое значение 1–1000).
2. Нажмите **Сгенерировать**. Создаются 5 тестовых пар ключей.
3. Нажмите **Финансировать** для перевода 0.01 ETH (газ) каждому
   избирателю.

4. Нажмите **Добавить избирателей в whitelist**.

### 4. Запуск голосования

Прокрутите до **Управление стадией**. Нажмите **Начать голосование**.
Стадия меняется на `ACTIVE`.

### 5. Подача голосов

Переключитесь на вкладку **Голосование**.

**Вариант A — Ручное голосование:**

1. Нажмите **Загрузить JSON** и выберите `data/exports/voters/voters.json`
   (или вставьте ключ в поле вручную).

2. Нажмите **Загрузить кандидатов**.
3. Выберите кандидата.
4. Нажмите **Проголосовать**.

Дождитесь тоста `Голос успешно отправлен`. Появятся TX Hash и
QR-квитанция.

**Вариант B — Массовое голосование (рекомендуется для первого запуска):**

1. В разделе "Массовое голосование (Тестирование)" нажмите
   **Из сессии**.

2. Подтвердите диалог.
3. Наблюдайте за логом: избиратели обрабатываются последовательно,
   каждый случайный голос фиксируется с TX hash.

### 6. Завершение голосования

Вернитесь на вкладку **Admin**. Нажмите **Завершить голосование**.
Стадия меняется на `FINISHED`.

### 7. Запуск аудита

Переключитесь на вкладку **Аудит**.

1. В комбобоксе **Режим аудита** выберите **Полный аудит**.
2. Нажмите **Запустить аудит**.

Таблица проверок безопасности заполнится статусами `PASSED` /
`FAILED` / `WARNING` для всех шести SEC-проверок. Таблица результатов
покажет количество голосов по кандидатам и победителя.

### 8. Экспорт результатов

Нажмите **JSON** или **CSV** на вкладке Аудит. Файлы сохраняются в
`data/exports/`.

---

## Что сохранилось

После завершения цикла:

| Артефакт | Расположение |
|---|---|
| Лог сессии        | `data/logs/active/session.log` |
| Экспорт избирателей | `data/exports/voters/voters.json` |
| Отчёты аудита     | `data/exports/audit_report.json` / `.csv` |
| QR-квитанция      | `data/exports/receipts/<tx_hash>.png` |
